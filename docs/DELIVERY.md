# DELIVERY - 规划型测试仓库

## 验收用途

本文档用于人工复核研究规划、设计规划、开发测试规划三类任务是否只输出计划，不越界执行研究、设计或开发。

## 交付物

| 任务 | 交付物 | 验收要点 |
| --- | --- | --- |
| 研究规划 | `plans/research-plan.md` | 包含研究任务、依赖、验收、并行策略和停止条件，不包含研究结论。 |
| 设计规划 | `plans/design-plan.md` | 包含设计任务、依赖、验收、并行策略和停止条件，不包含详细设计正文。 |
| 开发规划 | `plans/development-plan.md` | 包含开发任务、测试任务、依赖、验收命令、并行策略和停止条件，不实现代码。 |

## 验收命令

```bash
test -s plans/research-plan.md
test -s plans/design-plan.md
test -s plans/development-plan.md
```

## 人工验收清单

- 每个计划均使用简体中文。
- 每个计划均包含任务拆分、依赖关系、可并行批次、验收标准和停止条件。
- 研究规划不写研究结论。
- 设计规划不写 PRD/HLD/LLD 正文。
- 开发规划不修改源码或测试代码。
- 最终回复说明计划路径、执行顺序和主要阻塞条件。
