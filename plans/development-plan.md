# 任务质量仪表盘开发与测试任务规划

## 1. 规划边界

本计划用于调度“spec-executor 任务质量仪表盘”项目的**开发与测试任务**。规划输入为 `inputs/project-brief.md`、`docs/PRD.md`、`docs/HLD.md`、`docs/LLD.md` 以及上游 `plans/design-plan.md`（DD-01 至 DD-06 的详细设计任务产物）。

本任务只产出 `plans/development-plan.md`，**不实现代码、不编写测试代码、不修改 task package、不修改源码 / schema / 迁移 / 运行时脚本**。计划描述后续开发任务应当如何被拆分、排序、并行、验收与停止，并给出建议的验收命令；这些命令由后续实现任务在真实代码存在后执行，本任务不运行它们。

来源需求映射：
- `docs/PRD.md` FR-003：开发规划任务输出开发与测试计划。
- `docs/PRD.md` FR-004：所有计划必须包含任务、依赖、验收和并行策略。
- `docs/LLD.md`：计划必须包含依赖、可并行性、验收标准和阻塞条件。
- `docs/DELIVERY.md`：开发规划须包含开发任务、测试任务、依赖、验收命令、并行策略和停止条件，且不实现代码。

产品链路（来自 `inputs/project-brief.md`）：研究质量信号 → 设计数据模型与交互 → 实现 CLI 导出与 Markdown 报告。开发任务在设计任务（DD-01..DD-05）产物就绪后执行。

## 3. 测试任务拆分（测试策略）

测试策略遵循三层结构：单元测试、集成测试、端到端（CLI）测试；每个实现任务须先写测试（RED）再实现（GREEN）。以下为独立可验收的**测试任务**。

| 任务 ID | 测试任务名称 | 测试类型 | 测试职责 | 对应实现任务 |
| --- | --- | --- | --- | --- |
| TEST-01 | 指标口径单元测试 | 单元 | 覆盖 DD-01 指标计算、缺失数据处理、质量分层的正常与边界用例。 | IMPL-02 |
| TEST-02 | 数据模型与序列化单元测试 | 单元 | 覆盖 DD-02 数据对象构造、字段来源、序列化/反序列化与兼容策略。 | IMPL-03 |
| TEST-03 | CLI 流程集成测试 | 集成 | 覆盖 CLI 参数解析、读取/聚合/导出、成功与失败路径、退出码。 | IMPL-04 |
| TEST-04 | 报告生成快照/结构测试 | 集成 | 覆盖 Markdown 报告结构、质量状态呈现、空状态输出稳定性。 | IMPL-05 |
| TEST-05 | 仪表盘展示行为测试 | 集成 | 覆盖视图渲染、筛选/排序/钻取、质量状态映射与空状态。 | IMPL-06 |
| TEST-06 | 端到端 CLI 验收测试 | E2E | 从样例输入到 CLI 导出与 Markdown 报告的完整链路验收。 | IMPL-07 |

测试策略要点：
- **覆盖率目标**：核心模块单元测试覆盖率不低于 80%。
- **测试先行**：每个实现任务的验收依赖对应测试任务先通过。
- **测试数据**：仅使用可确认来源（brief/设计产物）派生的样例数据，不臆造字段。
- **隔离性**：单元测试不依赖真实文件系统之外的外部服务；网络在本任务包被禁用。

## 5. 并行策略（可并行批次）

| 批次 | 可并行任务 | 调度条件 | 并行注意事项 |
| --- | --- | --- | --- |
| B1 | IMPL-01 | 设计产物 DD-01..DD-05 就绪后启动。 | 骨架先行，统一构建/测试命令。 |
| B2 | IMPL-02 | IMPL-01 验收通过后启动。 | 指标口径先落地，避免下游臆造字段。 |
| B3 | IMPL-03 | IMPL-02 验收通过后启动。 | 数据契约是 CLI/报告/展示的共同基座。 |
| B4 | IMPL-04、IMPL-05、IMPL-06；TEST-01、TEST-02 | IMPL-03 验收通过后启动。 | IMPL-04/05/06 均只消费 IMPL-03 契约、互不共享文件与状态，可并行；TEST-01/02 针对已完成的 IMPL-02/03，可与 B4 实现并行。 |
| B5 | TEST-03、TEST-04、TEST-05 | 各自对应的 IMPL-04/05/06 验收通过后分别启动。 | 三个测试任务互不依赖，可并行；不得跨任务共享夹具状态。 |
| B6 | IMPL-07 | IMPL-04、IMPL-05、IMPL-06 全部验收通过后启动。 | 集成装配，统一错误/缺失数据边界。 |

并行安全约束：同一批次内任务不得共享未决设计决策、文件、schema、接口、迁移、状态、测试夹具或存在直接/间接依赖路径。

## 6. 验收命令与验收标准

### 6.1 单任务建议验收命令（由后续实现任务在真实代码存在后执行）

以下为**建议**验收命令模板；具体命令名由 IMPL-01 确立的工程栈决定，本计划不锁定语言。示例形式：

```bash
# 工程骨架（IMPL-01）
<build 命令>            # 构建通过
<lint 命令>            # 静态检查通过

# 单元 / 集成 / E2E（IMPL-02..IMPL-07 + TEST-01..TEST-06）
<test 命令> --unit         # 单元测试通过
<test 命令> --integration  # 集成测试通过
<test 命令> --e2e          # 端到端测试通过
<coverage 命令>            # 覆盖率 >= 80%

# CLI 导出与 Markdown 报告（IMPL-04/05, TEST-06）
<cli 命令> export --input <样例> --out report.md
test -s report.md          # 报告文件非空
```

### 6.2 本规划文件（plans/development-plan.md）的验收标准

对应 `tasks/development-planning/spec.yaml` 的 verify 命令，本计划文件须满足：

```bash
test -s plans/development-plan.md
grep -q "测试" plans/development-plan.md
grep -q "依赖" plans/development-plan.md
grep -q "验收" plans/development-plan.md
grep -q "并行" plans/development-plan.md
grep -q "停止" plans/development-plan.md
```

### 6.3 单任务验收要求

每个开发/测试任务的验收必须满足：
- 只产出各自职责范围内的代码/测试文件；不越界修改其他任务的文件或 task package。
- 明确来源引用（brief / PRD / HLD / LLD / 对应 DD-0x 设计产物）。
- 对应测试任务通过，覆盖成功路径与失败/缺失数据路径。
- 与上游任务的命名、字段、状态、导出契约保持一致；冲突时停止并要求修订上游设计。

## 7. 全局验收（ACC-01 最终交付验收任务）

ACC-01 依赖全部非延期实现任务（IMPL-01..IMPL-07）与全部测试任务（TEST-01..TEST-06），须确认：
- “质量信号 → 数据模型/导出契约 → CLI 导出 → Markdown 报告 / 仪表盘展示”链路完整可运行。
- 每个当前阶段需求（FR-003/FR-004 及 brief 三段链路）都有实现覆盖、显式延期或阻塞说明。
- 全量构建、单元/集成/E2E 测试通过，覆盖率达标（>= 80%）。
- CLI 能对样例输入产出非空 Markdown 报告，报告结构与 DD-04 一致。
- 产出一份命名验收报告（如 `docs/acceptance-report.md`）记录来源可追溯性、需求覆盖、DAG 完成、测试证据与风险处置。

## 8. 停止条件（阻塞条件）

任一任务遇到以下情况必须停止并记录阻塞原因：
- `inputs/project-brief.md`、`docs/PRD.md`、`docs/HLD.md`、`docs/LLD.md` 或上游 `plans/design-plan.md`（DD-0x）缺失，或不足以支撑该任务的实现边界。
- 上游设计任务（DD-01..DD-05）未完成、验收失败，或其输出与实现目标冲突（如质量信号命名、状态枚举、字段来源不一致）。
- 需要新增项目未确认的用户画像、指标、数据字段、CLI 参数、报告格式或运行环境事实。
- 上游实现任务未通过验收，或其输出契约与当前任务冲突。
- 任务需要修改 task package、manifest、schema、迁移、运行时脚本或其他非本任务职责的文件。
- 覆盖率无法达到 80%，或关键失败/缺失数据路径无法测试。
- 并行批次执行后出现字段命名、状态枚举、报告结构或展示语义冲突且无法通过依赖调整解决。

## 10. 风险与缓解

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| 质量信号定义不足或指标歧义（DD-01 未确认） | 指标实现与展示口径漂移 | 停止 IMPL-02 并回写 DD-01 开放问题；不臆造指标。 |
| 数据契约变更 | CLI/报告/展示字段分裂 | 以 IMPL-03 契约为单一来源；下游新增需求回写为 DD-02 设计问题。 |
| 缺失数据处理不一致 | 报告/展示出现空值或崩溃 | 在 DD-01/DD-05 统一缺失数据规则，TEST 覆盖空状态。 |
| 覆盖率不足 | 验收失败 | TDD 先行，B4/B5 并行补测；覆盖率纳入单任务验收。 |
| 并行任务冲突 | 集成阶段返工 | B4/B5 仅并行无依赖路径、无共享文件的任务；集成前做一致性检查。 |
| 工程栈未在设计中锁定 | 验收命令不可执行 | 由 IMPL-01 确立并记录构建/测试/覆盖率命令，回填 6.1 模板。 |
